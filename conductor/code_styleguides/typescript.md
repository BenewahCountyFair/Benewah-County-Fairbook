# TypeScript Style Guide

- Use `const` by default and `let` only when reassignment is required; never use `var`.
- Use ES modules and named exports.
- Prefer explicit, specific types over `any` or unsafe assertions.
- Use strict equality checks (`===` and `!==`).
- Use `UpperCamelCase` for types and classes; use `lowerCamelCase` for variables and functions.
- Use JSDoc only when it adds information beyond the TypeScript types.
- Follow the existing project style where it conflicts with this concise guide.
